  1. 식별자 (identifier)

변수 이름을 식별자라고 한다. 식별자는 어떤 값을 구별해서 식별할 수 있는 고유한 이름을 말한다**.** 식별자는 값이 아니라 메모리 주소를 기억하고 있다. 식별자가 값을 구별해서 식별한다는 것은 식별자가 기억하고 있는 메모리 주소를 통해 메모리 공간에 저장된 값에 접근할 수 있다는 의미다.

식별자라는 용어는 변수 이름에만 국한해서 사용하지 않는다. 예를 들어, 변수, 함수, 클래스 등의 이름은 모두 식별자다. 식별자인 변수 이름으로는 메모리 상에 존재하는 변수 값을 식별할 수 있고, 함수 이름으로는 메모리 상에 존재하는 함수(자바스크립트에서 함수는 값이다.)를 식별할 수 있다. 즉, 메모리 상에 존재하는 어떤 값을 식별 할 수 있는 이름은 모두 식별자라고 부른다.

Preview) 변수, 함수, 클래스 등의 이름과 같은 식별자는 네이밍 규칙을 준수해야 하며, 선언(declaration)에 의해 자바스크립트 엔진에 식별자의 존재를 알린다.

variable

  1. 변수(variable)

변수(variable)은 하나의 값을 저장하기 위해 확보한 메모리 공간 자체 또는 그 메모리 공간을 식별하기 위해 붙인 이름을 말한다.

변수는 하나의 값을 저장하기 위한 매커니즘이다. 여러 개의 값을 저장하려면 여러개의 변수를 사용해야 한다. 단 배열이나 객체같은 자료구조를 사용하면 관련이 있는 여러개의 값을 그룹화해서 하나의 값처럼 사용할 수 있다.

변수가 왜 필요한 것일까?

  1. 메모리 주소를 통해 값에 직접 접근하는 것은 치명적인 오류를 발생시킬 가능성이 높은 매우 위험한 일이다. 만약 실수로 운영체제가 사용하고 있는 값을 변경하면 시스템을 멈추게 하는 치명적인 오류가 발생할 수도 있다. 따라서 개발자의 직접적인 메모리 제어를 허용하지 않는다.
  1. 개발자가 직접적인 메모리 제어를 허용한다고 하더라도 문제가 있다. 값이 저장될 메모리 주소는 코드가 실행될 때 메모리 상황에 따라 임의로 결정된다. 코드가 실행되기 이전에는 값이 저장된 메모리 주소를 알 수 없으며, 알려 주지도 않는다.

메모리 공간에 저장된 값을 식별할 수 있는 고유한 이름(userId, userName, user, users)을 변수이름(변수명)이라 한다. 그리고 변수에 저장된 값을 변수 값이라고 한다.

Algorithm-1. Bubble sorting algorithm

버블정렬

- 두 인접한 데이터를 비교해서, 앞에있는 데이터가 뒤에 있는 데이터보다 크면, 자리를 바꾸는 정렬 알고리즘

Big O

  • Worst Case: O(2^n): 정렬이 하나도 안되어 있는 경우
  • Best Case: O(n): 이미 정렬이 되어 있는 경우

버블 정렬은 최악의 경우에 O(2^n)의 시간 복잡도를 가진다. 왜냐하면 각 자리를 n번 순회를 해야 하며 n번의 회전 동안에 요소의 개수만큼 또 순회를 해야하기 때문이다. 그러나 이미 정렬이 되어있는 경우에는 한 번의 순회로 정렬 여부를 알 수 있다.

< 장점 >

  • 버블 정렬은 in-place 알고리즘이기 때문에 메모리가 절약됨. “In - place”라는 것은 자료를 정렬할 때, 추가적인 메모리 공간이 필요한 것이 아니고 데이터가 저장된 그 공간 내에서 정렬을 한다는 뜻

< 단점 >

  • 버블 정렬의 단점은 자료의 갯수가 많아질수록 성능이 매우 떨어짐. 최악의 경우 O(2^n)이 소요되기 때문이다.

Stable vs Unstable

버블 정렬은 중복 데이터가 있을 경우 데이터의 위치를 교환하지 않고 지나가기 때문에 Stable한 정렬 여기서 stable이라는 뜻은 중복 데이터가 있는 경우에 기존 중복 데이터의 순서가 정렬이 다 끝난 이후에도 유지되는 정렬을 말한다.

Sorting Algorithm(정렬 알고리즘)

Sorting Algorithm(정렬 알고리즘)

무작위로 섞여 있는 데이터를 어떤 기준에 맞춰 정렬하는 알고리즘은 여러 가지가 있다. 정렬 알고리즘은 다양한 경우에 매우 유용하게 사용된다.

  1. 각종 데이터 목록을 정리하고 싶을 때
  2. 분포도의 중위값을 알아내고 싶을 때
  3. 데이터에서 중복값을 잡아내고 싶을 때
  4. 이진 탐색을 하고 싶을 때

자바스크립트에 sort()라는 메소드가 이미 존재하듯이, 모든 프로그래밍 언어에는 자체적인 정렬 메소드가 있다. 그럼에도 불구하고 우리가 정렬 알고리즘을 배워야 하는 이유는 시스템 정렬이 항상 좋은 퍼포먼스를 보장하지 않기 때문이다. 또한 내가 가진 데이터베이스의 양이나 상황에 따라 어떤 정렬을 사용하는 것이 좋을 지 달라지기 때문에 우리는 기본적으로 정렬 알고리즘을 반드시 알아야 한다!

Float - Mysterious Attribute

Float (알면 알수록 신기한 속성)


MDN문서를 통해 float의 정의부터 알아보자!

CSS 속성(property) float는 한 요소(element)가 보통 흐름(normal flow)으로부터 빠져 텍스트 및 인라인 요소가 그 주위를 감싸는 자기 컨테이너의 좌우측을 따라 배치되어야 함을 지정합니다.

float는 블록 레이아웃의 사용을 뜻하기 때문에, 일부 경우에 display 값의 계산값을 수정한다. Inline, table(row, column, cell 등등)의 계산값을 block으로 수정한다. 하지만 flex와 inline-flex는 효과가 없다.

source: MDN / Float


공부를 하면서 float의 흥미로운 점을 발견했다. 그건 블록끼리는 밀어내지만 그 안에 인라인 요소들은 감싼다는 특징이 있다. 왜이러한 현상이 일어날까? 왜이러한 현상이 일어날까?



Block Formatting Context / Inline Formatting

BFC, IFC를 통해 원리를 조금이나마 이해할 수 있었다. 컴퓨터가 HTML을 렌더링할 때, 정적인 요소를 그리는 방법을 Normal Flow라고 합니다.
Normal Flow는 박스 단위로 요소의 크기를 계산하고(box-sizing: content-box, border-box) 두가지의 배치 방식(BFC, IFC)으로 요소의 위치를 정해 그린다.

  • float 속성 해제하는 방법:
    • BFC: 요소를 세로로 배치하는 컨텍스트
    • IFC: 요소를 가로로 배치하는 컨텍스트


BFC는 요소가 세로 배치할 수 있도록 보이지 않는 범위을 만들어서 계산을 해주며, IFC는 요소가 가로 배치할 수 있도록 보이지 않는 범위을 만들어 계산해 알려주는 것이다.



Overflow: hidden; 의 특성
자식까지 커버해주는 BFC를 형성하지만, 자식의 자식까지 크기를 커버해주지 못한다.
자식의 자식까지 커버해주기 위해서는 바로 위 부모요소의 부모보다 커버린 자식의 width나 height 값을 없애면 된다.



Float는 자신의 부모 BFC가 끝날 때까지 존재하며, 위의 이미지에 있는 div태그 옆에 붙어있게 된다. 이를 막을 수 있는 방법은

  1. 형제 요소에 clear: both; 를 주거나
  2. 부모에 overflow: hidden; 을 주거나
  3. float된 자식들을 모두 묶어서 clearfix를 부모 클래스에 부여하면 된다.
